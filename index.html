Product Promotion Import & Approve Backoffice Tool

Custom membership provider / section

New World had reached the limit of the Umbraco member section. They had so many members that the alpha list would no longer expand as it had reached JSON length limit. Also the section had become increasingly hard to manage as the speed has reduced greatly such that it was taking approximately a minute to load or save members in the CMS.

I was required to create a new membership provider that implemented the ASP Membership and would also communicate new subscriptions and changes to members to their new shopping application system. I used TDD and created an integration test project to ensure the code would perform all actions required. Also the shopping system required the ability to send subscriptions and updates to the main application such that I had to create an API for that system to communicate with that mirrored their own.

Once the membership section was complete it was tested with heavy load and was capable of holding at least a million members without any noticeable degradation in performance.
